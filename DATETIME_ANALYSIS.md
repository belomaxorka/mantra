# Анализ реализации системы работы с датами и временем

## Текущая реализация

### ✅ Что реализовано

1. **Базовые функции для работы с датами:**
   - `now()` - текущее время в настроенном часовом поясе
   - `now_utc()` - текущее время в UTC
   - `format_date()` - форматирование с учетом часового пояса
   - `time_ago()` - относительное время
   - `get_timezones()` - список часовых поясов
   - `convert_timezone()` - конвертация между поясами

2. **Хранение в UTC:**
   - Все временные метки сохраняются в UTC
   - Отображение в настроенном часовом поясе

3. **Интеграция в админку:**
   - Dropdown с выбором часового пояса
   - Все даты в админке и на сайте используют `format_date()`

4. **Базовая локализация:**
   - Переводы для единиц времени (EN/RU)
   - Переводы для "назад" и "через"

## ❌ Критические недостатки

### 1. **Неправильная поддержка множественного числа**

#### Проблема:
```php
$unit = $value === 1 ? t('datetime.hour', 'hour') : t('datetime.hours', 'hours');
```

Это работает только для английского языка!

#### Русский язык имеет 3 формы:
- **1, 21, 31, 41...** → "час" (1 час, 21 час)
- **2-4, 22-24, 32-34...** → "часа" (2 часа, 3 часа, 24 часа)
- **5-20, 25-30, 35-40...** → "часов" (5 часов, 10 часов, 25 часов)
- **0** → "часов" (0 часов)

#### Текущий результат:
```
1 час назад   ✅ правильно
2 часов назад ❌ должно быть "2 часа назад"
5 часов назад ✅ правильно
21 часов назад ❌ должно быть "21 час назад"
```

#### Другие языки:
- **Польский:** 4 формы множественного числа
- **Арабский:** 6 форм множественного числа
- **Китайский:** нет множественного числа

### 2. **Отсутствие локализации форматов дат**

#### Проблема:
```php
format_date($date, 'F j, Y')  // "March 19, 2026"
```

Использует PHP `date()`, который:
- Выводит названия месяцев на английском (даже если locale установлен)
- Использует английский порядок (месяц-день-год)
- Не учитывает региональные форматы

#### Что нужно:
- **EN:** March 19, 2026
- **RU:** 19 марта 2026 г.
- **DE:** 19. März 2026
- **JP:** 2026年3月19日

### 3. **Жестко закодированный формат относительного времени**

#### Проблема:
```php
return sprintf($suffix, $value . ' ' . $unit);
// Результат: "2 hours ago"
```

Порядок слов фиксирован: `число + единица + суффикс`

#### Разные языки имеют разный порядок:
- **EN:** "2 hours ago"
- **RU:** "2 часа назад"
- **JP:** "2時間前" (число + единица + суффикс, без пробелов)
- **AR:** "منذ ساعتين" (суффикс + единица в особой форме)

## ⚠️ Средние недостатки

### 4. **Нет поддержки коротких форматов**

Отсутствуют:
- "только что" (just now)
- "минуту назад" (a minute ago)
- "сегодня в 14:30" (today at 2:30 PM)
- "вчера в 18:00" (yesterday at 6:00 PM)

### 5. **Нет парсинга дат**

Отсутствует функция для парсинга пользовательского ввода:
```php
parse_date('19.03.2026', 'd.m.Y')  // не существует
```

### 6. **Нет информации о часовом поясе**

Нет функций для:
- Получения аббревиатуры (PST, MSK, JST)
- Получения смещения (+03:00, -08:00)
- Отображения полного названия пояса

### 7. **Производительность get_timezones()**

Функция генерирует ~400 часовых поясов:
```php
DateTimeZone::listIdentifiers()  // возвращает все зоны
```

Лучше было бы:
- Группировать по регионам
- Показывать только популярные зоны
- Добавить поиск

### 8. **Нет миграции существующих данных**

Если в базе уже есть даты в серверном часовом поясе, они не будут корректно конвертированы в UTC.

## 💡 Рекомендации по улучшению

### Приоритет 1: Исправить множественное число

#### Вариант A: Добавить функцию pluralize()

```php
/**
 * Get plural form for Russian language
 *
 * @param int $n Number
 * @param array $forms [singular, few, many] e.g. ['час', 'часа', 'часов']
 * @return string
 */
function pluralize_ru($n, $forms) {
    $n = abs($n) % 100;
    $n1 = $n % 10;

    if ($n > 10 && $n < 20) {
        return $forms[2];  // 11-19 часов
    }
    if ($n1 > 1 && $n1 < 5) {
        return $forms[1];  // 2-4 часа
    }
    if ($n1 == 1) {
        return $forms[0];  // 1 час
    }
    return $forms[2];  // 0, 5-20, 25-30 часов
}

function pluralize($n, $key) {
    $locale = config('locale.default_language', 'en');

    if ($locale === 'ru') {
        $forms = t($key . '.forms', ['hour', 'hours', 'hours']);
        return pluralize_ru($n, $forms);
    }

    // English: simple singular/plural
    return $n === 1 ? t($key . '.singular') : t($key . '.plural');
}
```

#### Вариант B: Использовать ICU MessageFormat

Требует расширение PHP `intl`:

```php
$formatter = new MessageFormatter('ru_RU', '{count, plural, one{# час} few{# часа} many{# часов} other{# часов}}');
echo $formatter->format(['count' => 2]);  // "2 часа"
```

### Приоритет 2: Локализация форматов дат

```php
function format_date_localized($time, $format = 'medium') {
    if (!extension_loaded('intl')) {
        return format_date($time, 'Y-m-d H:i:s');
    }

    $locale = config('locale.default_language', 'en');
    $tz = config('locale.timezone', 'UTC');

    $formatter = new IntlDateFormatter(
        $locale,
        IntlDateFormatter::MEDIUM,
        IntlDateFormatter::SHORT,
        $tz
    );

    if ($time instanceof DateTime) {
        return $formatter->format($time);
    }

    return $formatter->format(strtotime($time));
}
```

```php
// Парсинг даты
function parse_date($dateString, $format = 'Y-m-d H:i:s') {
    $tz = config('locale.timezone', 'UTC');
    $timezone = new DateTimeZone($tz);
    return DateTime::createFromFormat($format, $dateString, $timezone);
}

// Информация о часовом поясе
function get_timezone_info($tz = null) {
    if ($tz === null) {
        $tz = config('locale.timezone', 'UTC');
    }

    $timezone = new DateTimeZone($tz);
    $now = new DateTime('now', $timezone);

    return [
        'name' => $tz,
        'offset' => $timezone->getOffset($now),
        'offset_string' => $now->format('P'),  // +03:00
        'abbreviation' => $now->format('T'),   // MSK
    ];
}

// Проверка валидности часового пояса
function is_valid_timezone($tz) {
    try {
        new DateTimeZone($tz);
        return true;
    } catch (Exception $e) {
        return false;
    }
}
```
