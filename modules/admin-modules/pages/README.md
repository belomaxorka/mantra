# Pages Admin Module

Модуль для управления статическими страницами в админ-панели Mantra CMS.

## Возможности

- **Редактор CKEditor 5** - современный WYSIWYG редактор с поддержкой:
  - Форматирование текста (жирный, курсив, подчеркивание, зачеркивание)
  - Заголовки (H1-H4)
  - Списки (маркированные и нумерованные)
  - Ссылки и изображения
  - Таблицы
  - Медиа-контент
  - Выравнивание текста
  - Цитаты и блоки кода
  - Редактирование исходного кода

- **Статусы публикации**
  - Draft (Черновик) - страница не отображается на сайте
  - Published (Опубликовано) - страница доступна публично

- **Постер страницы** - возможность указать URL изображения для страницы

- **Интеграция с навигацией**
  - Чекбокс "Show in navigation menu" - добавляет страницу в меню сайта
  - Поле "Navigation order" - определяет порядок отображения в меню (меньшее число = выше в списке)

## Использование

### Доступ к модулю

1. Войдите в админ-панель: `/admin`
2. В боковом меню выберите "Pages" (в разделе "Content")
3. Или используйте Quick Action "New Page" на дашборде

### Создание страницы

1. Нажмите "New Page"
2. Заполните поля:
   - **Title** (обязательно) - заголовок страницы
   - **Slug** - URL страницы (генерируется автоматически из заголовка)
   - **Content** - содержимое страницы (редактор CKEditor)
   - **Status** - Draft или Published
   - **Image** - URL постера страницы
   - **Show in navigation** - отображать в меню сайта
   - **Navigation order** - порядок в меню (по умолчанию 50)
3. Нажмите "Create Page"

### Редактирование страницы

1. В списке страниц нажмите кнопку "Edit" (карандаш)
2. Измените нужные поля
3. Нажмите "Update Page"

### Удаление страницы

1. В списке страниц нажмите кнопку "Delete" (корзина)
2. Подтвердите удаление

## Интеграция с темой

Страницы с включенным флагом `show_in_navigation` автоматически добавляются в навигационное меню сайта.

Это реализовано через отдельный модуль `pages` (не путать с `admin-modules/pages`), который:
- Загружается на всех страницах сайта (не только в админке)
- Подключается к хуку `theme.navigation`
- Добавляет опубликованные страницы с `show_in_navigation: true` в меню

Порядок отображения определяется полем `navigation_order` (меньшее число = выше в списке).

### Зависимости

Модуль `admin-modules/pages` зависит от модуля `pages`:
- В `module.json` указано: `"requires": ["admin", "pages"]`
- Если модуль `pages` отключен, навигационные настройки скрываются в интерфейсе редактирования
- Вместо них отображается информационное сообщение с предложением включить модуль `pages`

**Важно:** Модуль `pages` должен быть включен в `content/settings/config.json` → `modules.enabled`.

## Схема данных

Каждая страница хранится в `content/pages/{id}.json` со следующей структурой:

```json
{
  "_id": "page-slug",
  "title": "Page Title",
  "slug": "page-slug",
  "content": "<p>HTML content</p>",
  "status": "published",
  "image": "https://example.com/image.jpg",
  "show_in_navigation": true,
  "navigation_order": 10,
  "author": "Admin",
  "created_at": "2026-03-16 20:00:00",
  "updated_at": "2026-03-16 20:00:00",
  "schema_version": 1
}
```

## Маршруты

- `GET /admin/pages` - список страниц
- `GET /admin/pages/new` - форма создания
- `POST /admin/pages/new` - создание страницы
- `GET /admin/pages/edit/{id}` - форма редактирования
- `POST /admin/pages/edit/{id}` - обновление страницы
- `POST /admin/pages/delete/{id}` - удаление страницы

## Безопасность

- Все маршруты защищены middleware `requireAuth`
- CSRF-токены для всех POST-запросов
- Валидация и санитизация входных данных
- Защита от directory traversal в ID страниц
